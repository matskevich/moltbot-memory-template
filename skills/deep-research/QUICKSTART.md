# Deep Research — Быстрый старт

## Запуск

Когда вижу триггеры:
- "deep research"
- "исследуй все варианты"
- "найди все возможные подходы"
- "проведи глубокий анализ"

→ **Читаю SKILL.md** и следую методологии.

---

## Чек-лист

```
[ ] Фаза 1: Декомпозиция (5-10 мин)
    [ ] Разбить на аспекты
    [ ] 5-10 исследовательских вопросов
    [ ] Критерии оценки

[ ] Фаза 2: Широкий поиск (10-15 мин)
    [ ] 2-3 запроса на аспект
    [ ] web_search → топ-5 результатов
    [ ] Собрать ссылки

[ ] Фаза 3: Углублённый анализ (15-20 мин)
    [ ] web_fetch для топ-5-10 источников
    [ ] Выписать ключевые идеи
    [ ] Отметить противоречия

[ ] Фаза 4: Синтез (10 мин)
    [ ] Группировка паттернов
    [ ] Сравнительный анализ
    [ ] Пробелы

[ ] Фаза 5: Отчёт (5-10 мин)
    [ ] Markdown по шаблону
    [ ] Сохранить в ~/clawd/research/
    [ ] Обновить INDEX.md
```

---

## Шаблон для копирования

```bash
cp ~/clawd/skills/deep-research/scripts/template.md \
   ~/clawd/research/$(date +%Y-%m-%d)_название.md
```

---

## Пример workflow

```
USER: "Deep research: какие есть подходы к caching в веб-приложениях?"

CLAWD:
1. Читаю SKILL.md
2. Декомпозиция:
   - Client-side (browser cache, service workers)
   - Server-side (Redis, Memcached, in-memory)
   - CDN caching
   - Application-level (query caching, object caching)
   - Cache invalidation strategies
   
3. Исследовательские вопросы:
   - Какие типы кеширования существуют?
   - Как выбрать между Redis и Memcached?
   - Best practices для cache invalidation?
   - Как измерять эффективность кеша?
   - Типичные ошибки при кешировании?

4. Широкий поиск:
   web_search("web application caching strategies", count=10)
   web_search("Redis vs Memcached comparison", count=10)
   web_search("cache invalidation best practices", count=10)
   ...

5. Углублённый анализ топ-источников:
   web_fetch("https://...")
   ...

6. Синтез + отчёт → ~/clawd/research/2026-01-30_web-caching.md

7. Обновить INDEX.md
```

## Параметры по умолчанию

- **Время:** ~45-60 мин (полное исследование)
- **Источников:** 15-20 минимум
- **Глубокий анализ:** топ-5-10
- **Формат:** markdown по шаблону

---

## Адаптация под время

**Экспресс (20-30 мин):**
- 3-5 вопросов
- 1-2 запроса на аспект
- Топ-3 источника для анализа

**Углублённое (2-3 часа):**
- 10+ вопросов
- 5+ запросов на аспект
- Анализ 15-20 источников
- Дополнительные разделы (case studies, code examples)

---

## Мета

Этот skill — для **сложных, неочевидных задач**, где нужен comprehensive анализ.

Для простых вопросов — обычный web_search достаточен.
